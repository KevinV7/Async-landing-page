## Architecture

Single-page React portfolio site (no routing, no backend, no API calls). `src/App.jsx` renders a fixed sequence of section components inside `<main>`: `Navbar`, `Hero`, `TechStack`, `Projects`, `Experience`, `Contact`, `Footer`.

**All editable content lives in `src/data/portfolio.js`** — `profile` (name, role, contact links), `techStack`, `projects`, `experience`. Section components read from this file rather than hardcoding copy. When asked to update site content (name, projects, links, etc.), edit this file, not the components.

`Contact.jsx` has no backend — its form builds and opens a `mailto:` link with the message content.

### Design system (Neobrutalism)

Styling is Tailwind CSS 4 via the `@tailwindcss/vite` plugin (no `tailwind.config.js` — theme is defined with CSS `@theme` in `src/index.css`).

- Design tokens (`--color-cream`, `--color-acid`, `--color-punch`, `--color-splash`, `--color-focus`, `--shadow-brutal*`, `--radius-brutal`, fonts) are all defined in `src/index.css` under `@theme`. Change the look of the whole site here, not per-component.
- `src/components/ui/` holds the base primitives (`Button`, `Card`, `Badge`) that every section composes. They share the same visual vocabulary: thick black borders (`border-[3px] border-black`), hard offset shadows with no blur (`shadow-brutal`), and a variant prop mapping to one of the theme colors (`yellow`/`pink`/`cyan`/`white`).
- Interactive "press" affordance: buttons/cards translate toward their shadow on active/hover (`press-effect`, `lift-on-hover` classes) and lose the shadow, simulating depth. This is implemented with Tailwind utility classes directly in the component, not in CSS, except for the `prefers-reduced-motion` override in `index.css` which disables it.
- Accessibility constraints baked into the system: text is always black-on-light/saturated background for contrast; `:focus-visible` uses a dedicated blue outline (`--color-focus`) separate from the decorative black border; `prefers-reduced-motion: reduce` disables press/hover transforms site-wide. Preserve these when adding new interactive components.

When adding a new UI primitive, follow the existing pattern in `src/components/ui/`: a `variants` map keyed to theme colors, thick border + `shadow-brutal`, and props spread onto the root tag.
